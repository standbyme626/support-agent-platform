# ROADMAP

## Z3. 下一阶段里程碑

## M1. 生产化接入与可靠性（2-3 周）

- 补齐渠道签名校验、重放防护、幂等处理。
- 增加入口失败重试与死信策略。
- 风险：多渠道协议差异导致解析分支膨胀。
- 缓解：统一 adapter 合同测试和字段规范。

## M2. 工单策略与规则精细化（2-3 周）

- 扩展优先级/队列规则，支持更细粒度 SLA 策略。
- 强化 handoff 条件与接回机制。
- 风险：规则增长引发可维护性下降。
- 缓解：规则配置外置化 + 回归样例库。

## M3. 可观测与评估自动化（1-2 周）

- 建立固定演示样本与自动验收脚本。
- 增加 trace KPI（链路完整率、关键事件缺失率）。
- 风险：数据样本覆盖不足导致误判。
- 缓解：按故障类型持续扩容 seed 数据。

## M4. 部署与协作工程化（2 周）

- 增加容器化、CI/CD、环境分层发布流程。
- 补充运行手册中的回滚与值班流程。
- 风险：环境漂移导致“本地可用、线上失败”。
- 缓解：环境配置基线校验 + 发布前 smoke test。

## 持续约束

- 维持 `workflow-first, agent-assisted`。
- OpenClaw 仅做 ingress/session/routing。
- 前端后台仍不进入当前交付范围，除非里程碑显式变更。
