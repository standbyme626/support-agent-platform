# 升级5实施要求与验收测试规范

> 版本：v0.1（2026-03-13）  
> 适用范围：升级5分阶段实施（不改动 `升级5.md` 源文档）

---

## 0. 开工门（Go/No-Go）

### 0.1 必须确认

1. 仍遵循 `workflow-first, agent-assisted`，不改成 fully autonomous。
2. 高风险动作继续 HITL，可暂停、可恢复、可审计。
3. `API_CONTRACT_V1.md` 保持冻结；语义变更走 `API_CONTRACT_V2.md`。
4. 本轮只做“最小升级面”，不并行做数据库大迁移。
5. 必须采用“先边界后实现”的四步投喂顺序。

### 0.2 禁止项

1. 禁止把 `/close` 继续作为万能动作。
2. 禁止把 session 与 ticket 状态再次揉回同一对象。
3. 禁止跳过阶段门直接做 graph/agent 层实现。
4. 禁止绕过回归测试进入下一阶段。

---

## 1. 实施阶段与退出标准

## S1：建目录与空文件（不填复杂实现）

### 目标

创建升级5最小 10 文件骨架，仅含占位结构和文档头。

### 退出标准

1. 文件存在且可导入（Python 文件至少无语法错误）。
2. 不改变现有业务行为。

---

## S2：先填 4 个边界文件

### 范围

1. `API_CONTRACT_V2.md`
2. `app/domain/conversation/conversation_state.py`
3. `app/domain/ticket/ticket_workflow_state.py`
4. `app/application/session_service.py`

### 退出标准

1. v2 契约明确新增动作与兼容策略。
2. `ConversationState` 与 `TicketWorkflowState` 字段边界清晰，无交叉语义。
3. `session_service.py` 仅处理会话上下文，不写 ticket 生命周期规则。

---

## S3：再填 ticket 动作（限定范围）

### 范围

只改 `ticket_api.py` 相关实现，动作限定为：

1. `resolve`
2. `customer_confirm`
3. `operator_close`
4. `end_session`

### 退出标准

1. `/close` 不再承担多语义自动推断。
2. 四个动作事件类型、审计字段、状态迁移明确且可测。
3. v1 兼容入口行为被显式约束（可转发，不可扩展语义）。

---

## S4：最后填 graph + deep agent 入口

### 范围

1. `app/graph_runtime/intake_graph.py`
2. `app/agents/deep/ticket_investigation_agent.py`

### 退出标准

1. graph 仅负责编排，不吞并 ticket/session 规则。
2. deep agent 仅做分析建议，不直接执行高风险终态动作。

---

## 2. 测试与质量门禁

## 2.1 每阶段最小验证

1. `python -m pytest tests/integration/test_session_api.py -q`
2. `python -m pytest tests/integration/test_ticket_actions_api.py -q`
3. `python -m pytest tests/workflow/test_case_collab_workflow.py -q`

## 2.2 阶段收口建议

1. `python -m pytest tests/workflow -q`
2. `python -m pytest tests/regression -q`
3. `python -m pytest tests/integration -q`

## 2.3 全局门禁（如项目可用）

1. `make validate-structure`
2. `make lint`
3. `make typecheck`
4. `make test`
5. `make check`
6. `make acceptance`

---

## 3. 验收证据（每阶段必须留档）

1. 改动文件清单（新增/修改）。
2. 关键测试结果摘要（通过/失败与原因）。
3. 风险点与回滚策略。
4. 是否保持三条边界：
   - `/close` 非万能。
   - session/ticket 已拆边界。
   - 高风险动作仍 HITL。
