## 1. 升级原则（必须遵守）

1. **不推倒重做。**
   保留当前 `workflow-first, agent-assisted` 主骨架。([github.com](https://github.com/standbyme626/support-agent-platform) ([GitHub][2]))

2. **不把 OpenClaw 做重。**
   OpenClaw 继续只作为接入层 / session 层 / routing 层使用，不让它承接 ticket 业务规则。OpenClaw 官方也明确把 Gateway 定位为 `sessions / routing / channel connections`，且 routing 是 host-controlled 的确定性配置。([docs.openclaw.ai](https://docs.openclaw.ai/channels/channel-routing?utm_source=chatgpt.com) ([GitHub][1]))

3. **不转向 autonomy-first。**
   不做 AutoGPT 风格的开放式自循环。继续坚持：

   * 主干：固定 workflow
   * 局部：agent-assisted（分类、检索、摘要、推荐动作）
     这与 LangGraph 对 workflow 与 agent 的区分是一致的。([docs.langchain.com](https://docs.langchain.com/oss/python/langgraph/workflows-agents?utm_source=chatgpt.com) ([GitHub][2]))

4. **优先补“工单系统真实感”，不是补“花哨功能”。**
   优先改 ticket 数据模型、状态机、队列、指派、SLA、handoff、协同命令、trace，而不是前端或多余自主性。
